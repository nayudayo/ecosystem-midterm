Image Display

Chapter 4: Visualization and Quality in Digital Image Processing

Introduction to Image Display

Image Display in digital image processing refers to visualizing image data for human interpretation. It allows researchers and developers to inspect image properties and is crucial for applications like medical imaging, satellite analysis, and AI training.

Image Quality

Image quality is influenced by various attributes and is inherently subjective in nature.

Key Considerations

  • Quality is highly subjective - different viewers may have different opinions
  • Human vision generally prefers sharp and detailed images
  • Quality depends on spatial resolution and quantization

Image Data Types

uint8

Integer values from 0 to 255

  • Commonly used for 8-bit images
  • 0 = black, 255 = white
  • Standard format for many image files

double

Floating-point values typically scaled to [0,1]

  • Used for precise calculations
  • 0 = black, 1 = white
  • Common output type for processing functions

Working with Different Data Types

Handling Options

  1. Convert to uint8 before display
  2. Display matrix directly with appropriate scaling

Type Conversion Example


>> c = imread('caribou.tif');     % Read image as uint8
>> cd = double(c);                % Convert to double
>> imshow(c)                      % Display original
>> figure, imshow(cd);            % Display converted (appears black)

Image Scaling

When working with double-precision images, proper scaling is essential for correct display.

Scaling Example


>> cd = double(c);                % Convert to double
>> imshow(cd/255)                 % Scale to [0,1] range

Note: Division by 255 scales 8-bit values (0-255) to the range [0,1]

Important Considerations

  • Always check the data type of your image
  • Understand the expected value range for display
  • Use appropriate scaling for your data type
  • Consider the impact on image quality
