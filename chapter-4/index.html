Chapter 4: Visualization and Quality in Digital Image Processing

Introduction to Image Display

Image Display in digital image processing refers to visualizing image data for human interpretation. It allows researchers and developers to inspect image properties and is crucial for applications like medical imaging, satellite analysis, and AI training.

Key Applications

  • Medical Imaging Analysis
  • Satellite Data Processing
  • AI Model Training and Validation
  • Scientific Research Visualization

Image Quality

Image quality is influenced by various attributes and is inherently subjective in nature. The perception of quality depends on two main properties: spatial resolution and quantization.

Key Considerations

  • Quality is highly subjective - different viewers may have different opinions
  • Human vision generally prefers sharp and detailed images
  • Quality depends on spatial resolution and quantization

Quality Factors

  • Spatial Resolution: Determines the level of detail in the image
  • Quantization: Affects the precision of pixel values
  • Contrast: Influences the visibility of details
  • Sharpness: Impacts the clarity of edges and features

Image Data Types

Understanding image data types is crucial as they affect how MATLAB processes and displays images. Different MATLAB image processing commands may produce output matrices of varying types.

uint8

Integer values from 0 to 255

  • Commonly used for 8-bit images
  • 0 = black, 255 = white
  • Standard format for many image files
  • Most efficient for storage and display

Example Usage


c = imread('image.tif');  % Reads as uint8
imshow(c);               % Displays correctly

double

Floating-point values typically scaled to [0,1]

  • Used for precise calculations
  • 0 = black, 1 = white
  • Common output type for processing functions
  • Requires proper scaling for display

Example Usage


cd = double(c);         % Convert to double
imshow(cd/255);        % Scale for display

Key Differences

  • uint8 is optimized for storage and direct display
  • double provides more precision but requires scaling
  • Many MATLAB functions output double by default
  • Conversion between types may be necessary for proper display

Working with Different Data Types

When working with images in MATLAB, you often need to handle different data types. Understanding how to convert between types and display them correctly is essential.

Handling Options

  1. Convert to uint8 before display
    • Best for final output
    • Preserves standard image format
  2. Display matrix directly with appropriate scaling
    • Useful during processing
    • Requires understanding of value ranges

Type Conversion Example


>> c = imread('caribou.tif');     % Read image as uint8
>> cd = double(c);                % Convert to double
>> imshow(c)                      % Display original
>> figure, imshow(cd);            % Display converted (appears black)

Note: The second image appears black because double values need scaling for proper display

Image Scaling

When working with double-precision images, proper scaling is essential for correct display. The scaling process converts the numerical range of the image data to match the expected display range.

Scaling Example


>> cd = double(c);                % Convert to double
>> imshow(cd/255)                 % Scale to [0,1] range

Note: Division by 255 scales 8-bit values (0-255) to the range [0,1]

Important Considerations

  • Always check the data type of your image
  • Understand the expected value range for display
  • Use appropriate scaling for your data type
  • Consider the impact on image quality

Common Scaling Operations

  • uint8 to double: Divide by 255
  • double to uint8: Multiply by 255
  • Normalization: Scale to specific range
  • Custom scaling: Based on data requirements

imshow Function Details

The imshow function is versatile and supports various input formats and display options. It can display grayscale, RGB, binary, and indexed images with different parameters for customization.

Common Syntax Forms

  • imshow(I) - Displays grayscale image I
  • imshow(I, [low high]) - Displays with specified range
  • imshow(I, []) - Auto-scales display range
  • imshow(X, map) - Displays indexed image with colormap
  • imshow(filename) - Displays image from file
  • imshow(_, Name=Value) - Uses name-value pairs

Example Usage


% Read and display an image
img = imread('testdeer.jpg');
imshow(img, 'InitialMagnification', 'fit', 'Border', 'tight');

% Display with custom range
imshow(img, [0 255]);

% Auto-scale display
imshow(img, []);

Effects of Different Scaling Factors

The choice of scaling factor significantly affects the visual appearance of displayed images. Understanding these effects is crucial for proper image visualization.

Scaling Effects

  • Larger Divisor (e.g., /512): Results in lower pixel values (0 to 0.5 range). Image appears darker as even the brightest pixels become mid-grey.
  • Smaller Divisor (e.g., /128): Many pixel values exceed 1 and are clipped to white, resulting in an over-exposed appearance.
  • Optimal Divisor (255 for 8-bit): Provides proper scaling to the [0,1] range for correct display.

Scaling Examples


% Different scaling factors
imshow(cd/512)  % Darker image
imshow(cd/128)  % Over-exposed image
imshow(cd/255)  % Proper scaling

Working with Indexed Images

Indexed images use a colormap to define the colors in the image, allowing for effective storage and display of images with a limited number of colors.

Key Concepts

  • Images are stored as a matrix of indices into a colormap
  • Colormap is an m-by-3 matrix defining RGB triplets
  • Useful for images with limited color palettes
  • Can reduce memory usage and processing time

Example Usage


% Convert RGB to indexed image
[X, map] = rgb2ind(RGB, 256);
imshow(X, map);

% Display with different colormap
imshow(X, jet(256));

Image Resizing

The imresize function allows you to scale images up or down by specifying either a scaling factor or exact dimensions. By default, it uses bicubic interpolation for optimal quality.

Resizing Options

  • Scale Factor: Use a number between 0 and 1 to reduce size, greater than 1 to enlarge
  • Exact Size: Specify the desired number of rows and columns
  • Interpolation Methods: 'nearest', 'bilinear', 'bicubic' (default)
  • Antialiasing: Can be enabled or disabled for downsampling

Example Usage


% Read original image
A = imread('testdeer.jpg');

% Resize to 30% of original size
B = imresize(A, 0.3);

% Resize with specific dimensions
C = imresize(A, [256 256]);

% Resize with different interpolation
D = imresize(A, 0.5, 'Method', 'nearest');

% Display results
figure;
subplot(2,2,1); imshow(A); title('Original');
subplot(2,2,2); imshow(B); title('30% Scale');
subplot(2,2,3); imshow(C); title('256x256');
subplot(2,2,4); imshow(D); title('Nearest Neighbor');

Key Considerations

  • Downscaling may lose image details
  • Upscaling cannot create new information
  • Choice of interpolation method affects quality
  • Consider memory usage for large images
